# new-api 小白运行部署手册

这份手册按“第一次接触项目”的方式写。你可以先照着本地跑起来，再考虑正式部署。

## 1. 数据库怎么选

项目支持三种数据库：

| 数据库 | 适合场景 | 是否推荐生产 |
| --- | --- | --- |
| SQLite | 本机测试、临时体验 | 不推荐 |
| PostgreSQL | 正式上线、支付、代理、日志统计 | 推荐 |
| MySQL | 正式上线也可以，用的人多 | 可用 |

我建议你正式部署用 PostgreSQL。原因是稳定、事务能力好、适合支付订单、额度扣费、代理佣金这类数据。SQLite 只是一个 `.db` 文件，方便测试，但多人使用、并发充值、扣费统计时不适合生产。

## 2. 项目连接数据库的原理

后端通过环境变量 `SQL_DSN` 判断用什么数据库：

- 不设置 `SQL_DSN`：默认使用 SQLite。
- `SQL_DSN` 以 `postgres://` 或 `postgresql://` 开头：使用 PostgreSQL。
- `SQL_DSN` 是 MySQL 格式：使用 MySQL。

日志数据库可以单独设置 `LOG_SQL_DSN`。如果不设置，日志和主数据使用同一个数据库。

Redis 通过 `REDIS_CONN_STRING` 设置。Redis 不是数据库，但它能提升缓存、限流、同步性能，生产建议开启。

## 3. 推荐生产数据库配置

### PostgreSQL 推荐写法

```env
SQL_DSN=postgresql://newapi_user:你的强密码@127.0.0.1:5432/newapi
REDIS_CONN_STRING=redis://:你的Redis密码@127.0.0.1:6379/0
TZ=Asia/Shanghai
ERROR_LOG_ENABLED=true
BATCH_UPDATE_ENABLED=true
SESSION_SECRET=换成一串很长的随机字符串
```

如果 PostgreSQL 是 Docker Compose 里的服务，主机名不是 `127.0.0.1`，而是服务名，例如：

```env
SQL_DSN=postgresql://root:123456@postgres:5432/new-api
```

### MySQL 写法

```env
SQL_DSN=newapi_user:你的强密码@tcp(127.0.0.1:3306)/newapi?charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
REDIS_CONN_STRING=redis://:你的Redis密码@127.0.0.1:6379/0
TZ=Asia/Shanghai
ERROR_LOG_ENABLED=true
BATCH_UPDATE_ENABLED=true
SESSION_SECRET=换成一串很长的随机字符串
```

MySQL 数据库字符集必须用 `utf8mb4`，否则中文、表情、日志内容可能出问题。

### SQLite 写法

不设置 `SQL_DSN` 就会用 SQLite。也可以指定文件路径：

```env
SQLITE_PATH=D:\newapi\one-api.db
```

SQLite 只建议本地测试。

## 4. 最简单的 Docker 运行方式

先确认 Docker Desktop 已启动。你之前遇到的错误：

```text
failed to connect to the docker API at npipe:////./pipe/dockerDesktopLinuxEngine
```

意思是 Docker Desktop 没打开，或者 Docker 后台服务没启动。

### 启动步骤

在 PowerShell 里执行：

```powershell
cd D:\newapi
docker compose up -d --build
```

启动成功后打开：

```text
http://localhost:3000
```

查看容器状态：

```powershell
docker compose ps
```

查看后端日志：

```powershell
docker compose logs -f new-api
```

停止：

```powershell
docker compose down
```

注意：`docker compose down` 不会删除数据库卷。不要随便执行 `docker compose down -v`，它会删除数据库数据。

## 5. 开发环境运行方式

开发时后端用 Docker 跑，前端单独跑。

### 1. 启动后端、PostgreSQL、Redis

```powershell
cd D:\newapi
docker compose -f docker-compose.dev.yml up -d
```

如果提示 Docker API 连接失败，先打开 Docker Desktop。

### 2. 启动前端

项目推荐 Bun，但你电脑之前没有 Bun。可以先用 npm：

```powershell
cd D:\newapi\web\default
npm install
npm run dev
```

前端一般打开：

```text
http://localhost:3001
```

后端 API 是：

```text
http://localhost:3000
```

### 3. 如果你想安装 Bun

PowerShell 执行：

```powershell
powershell -c "irm bun.sh/install.ps1 | iex"
```

安装后重新打开 PowerShell，再执行：

```powershell
bun --version
cd D:\newapi\web\default
bun install
bun run dev
```

## 6. 不用 Docker，本机直接运行后端

如果你已经有 Go 环境：

```powershell
cd D:\newapi
go run .
```

默认会使用 SQLite。正式环境不建议这样直接跑，建议用 Docker 或 systemd/进程管理器托管。

如果要连接 PostgreSQL：

```powershell
$env:SQL_DSN="postgresql://newapi_user:你的强密码@127.0.0.1:5432/newapi"
$env:REDIS_CONN_STRING="redis://:你的Redis密码@127.0.0.1:6379/0"
$env:SESSION_SECRET="换成一串很长的随机字符串"
go run .
```

## 7. 生产部署建议流程

### 第一步：准备服务器

推荐配置：

- 2 核 CPU 起步
- 4GB 内存起步
- 40GB 以上磁盘
- Ubuntu 22.04 或 Debian 12

### 第二步：安装 Docker 和 Docker Compose

装好后验证：

```bash
docker version
docker compose version
```

### 第三步：修改 `docker-compose.yml`

至少要改这些默认密码：

```yaml
POSTGRES_PASSWORD: 123456
REDIS password: 123456
SQL_DSN=postgresql://root:123456@postgres:5432/new-api
REDIS_CONN_STRING=redis://:123456@redis:6379
SESSION_SECRET=random_string
```

把 `123456` 改成强密码，`SESSION_SECRET` 改成随机长字符串。

示例：

```yaml
environment:
  - SQL_DSN=postgresql://root:你的数据库强密码@postgres:5432/new-api
  - REDIS_CONN_STRING=redis://:你的Redis强密码@redis:6379
  - TZ=Asia/Shanghai
  - ERROR_LOG_ENABLED=true
  - BATCH_UPDATE_ENABLED=true
  - SESSION_SECRET=这里换成至少32位随机字符串
```

PostgreSQL 服务也要同步改：

```yaml
postgres:
  environment:
    POSTGRES_USER: root
    POSTGRES_PASSWORD: 你的数据库强密码
    POSTGRES_DB: new-api
```

Redis 服务也要同步改：

```yaml
redis:
  command: ["redis-server", "--requirepass", "你的Redis强密码"]
```

### 第四步：启动

```bash
docker compose up -d --build
```

查看状态：

```bash
docker compose ps
docker compose logs -f new-api
```

### 第五步：绑定域名和反向代理

推荐用 Nginx、宝塔、1Panel、Nginx Proxy Manager 任选一个。

反代目标：

```text
http://127.0.0.1:3000
```

必须配置 HTTPS，尤其你有微信支付回调、登录、API Key。

## 8. 数据库备份

生产环境一定要备份。

### PostgreSQL 备份

如果使用 Docker Compose 里的 PostgreSQL：

```bash
docker exec -t postgres pg_dump -U root new-api > newapi_backup.sql
```

恢复：

```bash
docker exec -i postgres psql -U root new-api < newapi_backup.sql
```

### SQLite 备份

直接复制 `.db` 文件即可，例如：

```powershell
Copy-Item D:\newapi\one-api.db D:\backup\one-api.db
```

但备份前最好先停服务，避免文件正在写入。

## 9. 支付上线前必须检查

因为平台涉及充值，正式上线前务必检查：

1. 后台支付商户号、密钥必须是你自己的。
2. 支付回调地址必须是公网 HTTPS 域名。
3. 不要把测试支付密钥放到生产。
4. 不要公开数据库端口。
5. 不要公开 Redis 端口。
6. 定期核对平台充值订单和微信/支付平台真实到账订单。

## 10. 常见问题

### Docker 报 npipe/dockerDesktopLinuxEngine 找不到

原因：Docker Desktop 没启动。

解决：

1. 打开 Docker Desktop。
2. 等左下角显示 Docker 正常运行。
3. 重新执行 `docker compose up -d --build`。

### bun 不是命令

原因：电脑没安装 Bun。

解决：先用 npm 运行：

```powershell
cd D:\newapi\web\default
npm install
npm run dev
```

### 端口被占用

如果 `3000` 被占用，改 `docker-compose.yml`：

```yaml
ports:
  - "3002:3000"
```

外部访问就变成：

```text
http://localhost:3002
```

### 后台打不开

先看容器：

```powershell
docker compose ps
```

再看日志：

```powershell
docker compose logs -f new-api
```

如果日志里是数据库连接失败，重点检查 `SQL_DSN`、数据库密码、数据库服务名。

## 11. 给你的推荐方案

你现在这个平台已经涉及：

- 用户注册
- API 额度扣费
- 在线充值
- 代理佣金
- 模型价格
- 使用日志

所以推荐：

```text
生产数据库：PostgreSQL
缓存：Redis
部署方式：Docker Compose
访问方式：域名 + HTTPS + 反向代理到 3000
备份：每天自动备份 PostgreSQL
```

本地开发可以继续用：

```text
docker-compose.dev.yml + PostgreSQL + Redis + 前端 npm run dev
```
